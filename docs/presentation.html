"Next Generation AI Agents" Challenge

The Problem

  • Manual triage wastes hours. On-call engineers read reports, assess severity, copy-paste into ticket systems, and chase down context.
  • Duplicate tickets pile up. Without semantic dedup, the same outage spawns 5 tickets across reporters.
  • Screenshots go unread. Error screenshots sit in reports as opaque attachments — nobody extracts the stack traces.
  • Reporters are left in the dark. "Is anyone working on this?" — the most common follow-up on every incident.
  • No audit trail for triage decisions. Severity was assigned by gut feel; no structured record exists.

Our Solution

  • Zero-touch triage. Submit via GitHub issue or API → fully triaged ticket in Linear in seconds.
  • 7 specialized agents working in a deterministic pipeline — security, vision, triage, dedup, automation.
  • Multimodal analysis. Claude Vision reads screenshots, extracts error codes, and feeds them into triage.
  • Semantic dedup. LLM compares new incidents against existing tickets — same root cause = same ticket.
  • Full-loop notifications. GitHub comments, email — reporters know the status at every step.

Security & Guardrails

🛡 Prompt Injection Defense

Stage 1 — Regex (~1ms): 5 pattern categories catch obvious templates.
Stage 2 — LLM (~1-2s): Claude semantic analysis catches paraphrased, multilingual, obfuscated attacks.

hard-block: secret_exfil soft: sanitize + continue

🌐 SSRF Prevention

Image URLs validated against strict host allowlist before Claude Vision. Only GitHub CDN domains allowed. Blocked URLs never trigger external requests.

allowlist-based

🔑 Webhook Signature Verification

GitHub: HMAC-SHA256 via X-Hub-Signature-256.
Linear: HMAC-SHA256 + timestamp freshness (rejects >5 min old — replay defense).

hmac-sha256

📋 Audit Trail

Every prompt injection attempt is logged to prompt_injection_log in PostgreSQL: source, reporter, raw input, flags, blocked status, timestamp.

full traceability

Lessons Learned

1

Fallbacks are essential, not optional

Non-LLM fallback for every agent made the system testable and resilient. API rate limits hit during dev — fallbacks kept the demo working.

2

Two-tier guardrails hit the sweet spot

Regex alone misses paraphrased attacks. LLM alone is slow and expensive. The combination: fast floor + accurate ceiling.

3

Structured logging pays for itself immediately

JSON logs with consistent event names made debugging the multi-agent pipeline dramatically easier.

4

OpenTelemetry auto-instrumentation is magic

One line: AnthropicInstrumentor().instrument() — every LLM call traced with full prompt/response.

5

Stateless agents simplify everything

No shared memory, no coordination, no race conditions. Context flows through function arguments.

6

Hackathon scope discipline matters

Cut frontend, metrics, and CI early. Focus on pipeline quality, observability, and security — what judges evaluate.
